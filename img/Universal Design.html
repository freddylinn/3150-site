How Do We Achieve Universal Design on the Web?

HTML5 and CSS

HTML5 and CSS are core web technologies that come with a diverse set of tools to achieve universal design. The introduction of HTML5 in 2008 revolutionized semantic HTML. MDN Web Docs explains semantic HTML via an h1 tag: "In HTML, for example, the h1 element is a semantic element, which gives the text it wraps around the role (or meaning) of 'a top level heading on your page." HTML defines the content itself. An important idea to keep in mind with HTML5 is to be as specific as possible when structuring content. A common pitfall is that developers only use the div or span tags for content, but these have no semantic value. Developers can use the section tag to define sections, h2 or h3 tags to define section headers, p tags to define paragraphs, and so much more. A comprehensive list can be found here. Proper semantic tags help screen readers understand a website and navigate it accordingly.

CSS handles the styling of content and is crucial for universal design. CSS can make websites mobile-first and responsive, meaning you can define styles for mobile devices initially and then use media queries to define styles for desktop devices. This allows for different styling depending on screen size so that designers can provide smooth experiences regardless of device. Designers can also take advantage of CSS Grid and Flexbox to perfect their layouts. Color and font controls are imperative for providing proper contrast and sizing so that users with vision impairments like colorblindness or blurry vision can clearly see the content. The "em" and "rem" units size text according to a user's font preferences. You can also set certain items to only be read by screen readers, or to be ignored by screen readers.

Validation Tools

Validation tools can help spot errors in code that would go against best practices in accessibility. They can test the HTML itself, color contrast, semantics, and more. Certain validation tools will even give advice on how to improve the accessibility of your site on top of pointing out errors. A list is provided below along with their strengths.

  • W3C HTML Validator
  • Google Lighthouse (comprehensive)
  • WAVE (contrast and structuring errors)
